## 四、系统全景——5 个维度

整个系统的产品空间用 5 维坐标系描述。当前进度与终态如下：

| 轴 | 含义 | 当前进度 | 终态 |
|---|---|---|---|
| **知识类型** | 失败经验 / 成功经验 / 前沿 wiki / 品味 (taste) | ✅ 失败 + ✅ wiki（30+ 条）、⏳ 成功在累积 | 四类全覆盖 |
| **知识来源** | 个人 / 团队 / 互联网 | ✅ 个人（51 条）+ ✅ 全局（22 条）+ ⏳ wiki 源 5 个 | 三源融合 |
| **激活方式** | 实时拦截（被动）+ 对话注入（主动） | ✅ PreToolUse 被动拦截 + ✅ UserPromptSubmit 主动注入 | 主动 + 被动 |
| **工具范围** | Claude Code / Cursor / Windsurf / MCP 通用 | ✅ Claude Code | 多工具（Phase 6） |
| **用户范围** | 单人本地 / 团队共享 | ✅ 单人本地 + ✅ 陌生人可自行安装（Phase 3 完成）+ ⚠️ 团队共享仅 partial/status 可见 | 团队共享（Phase 4：transport/privacy/review gates 完成后） |

> **阅读提示**：本系统不在所有轴上铺摊子，而是按 Phase 把某一个角落做到无可挑剔再推进。当前已经把"个人 × 失败经验 × 被动 × Claude Code × 本地"做到 benchmark-ready，并开启了"wiki + 主动注入"+"陌生人自安装"两个增量轴。
