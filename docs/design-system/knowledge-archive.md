# Case C: Knowledge Archive

## Role

Knowledge Archive is the docs, trust, case study, and rule-governance direction.

It should explain where rules came from, how they changed, why they are safe to
share, and how a new team member inherits old agent experience.

## One-Screen Promise

一次踩坑，变成团队 agent 的长期经验。

## Aesthetic

Knowledge Archive is an engineering archive, not a wiki. It should feel like
an audit-friendly record of mistakes, rule lineage, evidence, and handoff.

It should be calm, readable, and source-first.

## Layout

Use documentation-forward product surfaces:

- left: clear explanation and navigation
- right: archive surface with tabs
- vertical timeline: learned -> compiled -> published -> loaded -> verified

Suggested tabs:

```text
Rules | Evidence | Sessions | Handoff | Privacy
```

## Typography

- Primary: Atkinson Hyperlegible Next feeling.
- Code and records: Atkinson Hyperlegible Mono or Iosevka Term feeling.
- Chinese fallback: Noto Sans SC.

The goal is fast reading and low ambiguity.

## Color

```text
Background: #FAFAF9
Text:       #111827
Slate:      #334155
Surface:    #F1F5F9
Primary:    #155E75
Rule:       #0F766E
Archived:   #6B7280
Warning:    #A16207
Danger:     #991B1B
Info:       #1D4ED8
```

Use muted archival colors. Keep danger visible but not dominant.

## Components

- Rule lineage timeline.
- Source transcript snippet.
- Confidence/tier history.
- `AGENTS.md` or `CLAUDE.md` compiled output preview.
- Evidence bundle reference.
- Privacy boundary diagram.
- Before/after case study.
- "What we do not store" list.

## Motion

Use calm interaction:

- timeline reveal
- evidence expand/collapse
- source-to-rule line highlight
- version comparison
- archived/new state transition

Avoid immersive animation. This direction is about trust and traceability.

## Safe Choices

- Source-first documentation builds confidence.
- Rule lineage reduces fear that TeamAgent invents or hides policy.
- Privacy sections help avoid employee-monitoring concerns.

## Creative Risk

Use "接力棒" as the team handoff metaphor: one agent's learned rule becomes the
next agent's starting context. Pair it with actual rule evidence so it does not
turn into soft knowledge-management copy.

## Best Use

Use this case for:

- docs landing pages
- security and privacy pages
- trust center
- case studies
- rule detail pages
- team onboarding material
