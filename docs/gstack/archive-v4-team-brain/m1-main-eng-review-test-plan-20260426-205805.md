# Test Plan
Generated by /plan-eng-review on 2026-04-26
Branch: main
Repo: V4teamBrain plus insights-share plugin

## Affected Pages/Routes
- Agent-controlled Web QA: use `/browse` and `/qa` against the supplied target URL; verify screenshots, console errors, network evidence, and report output under `.gstack/qa-reports/`.
- Human-controlled UAT: use an observed tmux/browser run directory; verify `uat-events.jsonl`, pane transcripts, screenshots, optional browser trace, and review report.
- Plugin terminal QA: use `~/projects/testV4team/uat.sh` and `docs/observed-pipeline-walkthrough.md`; verify fresh tmux behavior and post-run artifacts.

## Key Interactions to Verify
- Agent lane can start from a URL, click/fill/navigate, capture screenshots, and write a report without human input.
- Human lane can record a person-driven session without taking control away from the person.
- Agent post-review can read only the manifest and evidence paths, then classify pass/fail/blocker with repro steps.
- Both lanes emit the same normalized outcome fields: actor, action, expected, observed, evidence, timestamp, verdict.

## Edge Cases
- Missing screenshot or missing transcript path should mark a step as `evidence_missing`, not silently pass.
- A human run that never states expected outcome should be reviewable but flagged as low-confidence.
- Browser URL that points to an unrelated local service, such as macOS AirTunes on `localhost:5000`, should be excluded from product scoring.
- Trace files may contain sensitive text; evidence collection must run the existing PII filter before any shared upload.
- Long-running tmux UAT step 41 may be quiet while `tests/run.sh` completes; reviewers should look for the final `PASS=97 FAIL=0` and `114/114`.

## Critical Paths
- `/qa` lane: target URL -> browse actions -> screenshot/console/network evidence -> QA report -> baseline JSON.
- Human lane: user operates browser or terminal -> recorder writes transcript/evidence -> manifest index -> agent post-review -> QA report.
- Plugin lane: fresh tmux -> `~/projects/testV4team/uat.sh` -> full log + tail log -> report-only QA summary.
