# Test Plan
Generated by /plan-eng-review on 2026-04-26
Branch: main
Repo: V4teamBrain plus insights-share plugin

## Affected Pages/Routes
- Agent-controlled web lane: use `/browse` and `/qa` against the target URL, capture screenshots, console errors, network data, DOM snapshots, and final QA report.
- Human-controlled web lane: capture a real user session into one run directory, then let agents review the normalized manifest and linked evidence.
- Terminal/plugin lane: keep `~/projects/testV4team/uat.sh` as the existing fresh tmux UAT gate for non-Web plugin behavior.

## Key Interactions to Verify
- Agent can drive the same browser flow end-to-end without human input.
- Human can drive the browser normally while the recorder captures action, time, URL, screenshot, console, network, DOM, and optional video.
- Agent can post-review human trace without relying on video as the primary evidence.
- `/qa-only` can consume the same artifact shape for both lanes and produce one report.

## Edge Cases
- Human closes the browser, switches tab, refreshes mid-flow, or abandons an action.
- Sensitive fields appear in screenshots, console logs, network payloads, or typed text and must be redacted before storage.
- Console/network errors occur after an action but the UI still appears successful.
- Recorder crashes halfway through and must leave a partial manifest with a clear incomplete status.
- Agent-controlled and human-controlled runs produce different results for the same flow.

## Critical Paths
- Agent lane: target URL -> `/browse` actions -> evidence folder -> `/qa` report.
- Human lane: recorder start -> human actions -> `uat-events.jsonl` -> evidence folder -> agent post-review report.
- Unified report: both lanes normalize into the same manifest contract, so QA can compare them instead of treating them as unrelated evidence.
