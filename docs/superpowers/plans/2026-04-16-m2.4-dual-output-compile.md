**Goal:** 实现 TeamAgent Phase 2 M2.4 —— **双出口编译**。现有 `MarkdownCompiler` 只按行数裁剪、只看 `status=active`，没有 tier 过滤也没有 token 预算。M2.4 把它升级到 "canonical+ top-N + token budget ≤ 2000"，并新增 **Skill 出口**（`~/.claude/skills/teamagent/<id>/SKILL.md`，stable+ 规则自动导出，降级自动删除）。同时把 compile 从 analyze/pitfall 的副作用里拎出来，作为 `teamagent compile` 一等命令。

**Architecture:**
- **Functional Core 不破**：token 计数、tier 过滤、Skill MD 生成都是纯函数；IO（写 CLAUDE.md / 写 skill 目录 / 删 skill 文件）在 adapter 层。
- **Port 先行**：新增 `SkillCompiler` port（`compile(entries): SkillArtifact[]` + `write(artifacts)` + `cleanup(demotedIds)`）。对 `Compiler<T>` port 的既有 `compile(entries): T` 不改签名，改 `compileMarkdownBlock` 的 **options** 扩展（向后兼容，新参数走默认值）。
- **Tier 过滤放在 compile 函数内部而非调用方**，让契约可测（"输入混合 tier 的 entries，输出只含 canonical+"）——这是 advisor 意见 + 调用方不会漏过滤的保险。
- **降级删 skill 走 bus 事件**：calibration-pipeline-v2 在 tier transition 往 stable 以下（或 dormant）走时 emit `compile.skill_should_remove`；skill adapter 订阅该事件做 `fs.unlink`。避免在 core 里 import fs，也让多个出口将来可并行订阅。
- **Token 预算用 js-tiktoken**（已是 `packages/cli` 依赖）。把它提到 `packages/adapters` 的 token-counter，core 接受一个 `countTokens(str)` 回调注入（保持 core 无 IO、无 WASM / wasm-bindgen 依赖）。
- **stuck-in-promotion CLI** 捎带在 Task 12 落地 —— M2.3 plan 里标记为 M2.4 新命令，给被 L1/L2 反复挡在 probation 的规则一个可见性出口。做法：`teamagent stats --stuck-in-promotion` 列出 `tier_transition=null && reason_for_no_transition 含 "l1_blocked"/"l2_blocked"` 的规则 + 最近一次被挡的原因。

**Tech Stack:** TypeScript、Vitest、js-tiktoken（已在 `packages/cli/package.json`，M2.4 下沉到 `packages/adapters`）、pnpm workspace、zod（Skill frontmatter schema）。

**关键决策（已确认 / 可挑战）：**

1. **CLAUDE.md 预算 2000 tokens** 是硬上限（设计 §八），不是软警告。超限时按 `scoreEntry` 降序塞，塞不下就截断 + 在 footer 标注 `（… 还有 N 条 canonical+ 规则因 token 预算未显示，见 teamagent compile --dry-run）`。
2. **CLAUDE.md tier 过滤 = canonical + enforced**（设计 §4.4）。stable 进 Skills 不进 CLAUDE.md。probation/experimental 两端都不进。dormant 两端都不进。
3. **Skills 路径固定在 `~/.claude/skills/teamagent/<rule-id>/SKILL.md`**（设计 §5.6）。不走 project-local，因为 Claude Code skills 默认只扫 user 级目录。TEAMAGENT_SKILLS_DIR 环境变量可覆盖（dogfood / 测试用）。
4. **降级删 skill 用 `fs.unlink + ignore ENOENT`**，不做回收站，不保留 orphan 目录。一条规则的 skill 目录从 stable 跌到 probation → 目录整个 `rm -rf`；重新晋升则重新写。幂等。
5. **复用 `Compiler<T>` port 而非新增独立 port** 只对 markdown。Skill 引入**新 port** `SkillCompiler`（签名不同：产出是**多个文件**而不是一个字符串，既有 `Compiler<T>` 表达不出）。
6. **token 计数放 adapter 层，不放 core**。理由：js-tiktoken 的 encoder 初始化有 IO（读词表），core 保持"纯函数 + 注入 countTokens"。compile 函数签名加一个 `countTokens?: (s: string) => number`，缺省用 `Math.ceil(s.length / 3.5)` 粗估，让 core 测试不依赖 js-tiktoken。
7. **`teamagent compile` 作为一等命令**。analyze / pitfall / calibrate 原先各自 `new MarkdownCompiler().writeToFile(entries)` 的调用改为 `runCompile(deps)` 统一入口（**同时**输出 CLAUDE.md + Skills）。CLI 子命令做 dry-run 预览、force 重建等。
8. **stuck-in-promotion** 不是独立命令，挂 `teamagent stats --stuck-in-promotion`（Task 12）。理由：stats 已是"看状态"的入口，不必再开新命令面。

import { encodingForModel, getEncoding } from "js-tiktoken";

let cached: ReturnType<typeof getEncoding> | null = null;

/** 懒加载 cl100k_base encoder，返回一个 countTokens(s) 函数。线程安全由 node single-thread 保证。 */
export function createTiktokenCounter(): (s: string) => number {
  return (s: string) => {
    if (!cached) cached = getEncoding("cl100k_base");
    return cached.encode(s).length;
  };
}

import { createTiktokenCounter } from "../token-counter/index.js";

const DEFAULT_TIER_FILTER = ["canonical", "enforced"] as const;
const DEFAULT_TOKEN_BUDGET = 2000;

function resolveOptionsFromEnv(): CompileMarkdownOptions {
  const out: CompileMarkdownOptions = {
    tierFilter: DEFAULT_TIER_FILTER,
    tokenBudget: DEFAULT_TOKEN_BUDGET,
    countTokens: createTiktokenCounter(),
  };
  const rawBudget = process.env.TEAMAGENT_CLAUDE_MD_TOKEN_BUDGET;
  if (rawBudget) {
    const n = parseInt(rawBudget, 10);
    if (Number.isFinite(n) && n > 0) out.tokenBudget = n;
  }
  const rawLimit = process.env.TEAMAGENT_CLAUDE_MD_LIMIT;
  if (rawLimit) {
    const n = parseInt(rawLimit, 10);
    if (Number.isFinite(n) && n > 0) out.limit = n;
  }
  return out;
}

```typescript
import type { KnowledgeEntry } from "@teamagent/types";

/** 一条规则编译出的 skill artifact。Adapter 负责写到 skills 目录。 */
export interface SkillArtifact {
  ruleId: string;
  /** 目录相对路径（如 "my-rule-id"）；adapter 会拼到 SKILLS_DIR 下 */
  dirname: string;
  /** SKILL.md 文件内容（含 YAML frontmatter + body） */
  skillMd: string;
}

export interface SkillCompiler {
  /** 纯：把 entries 过滤 + 编译成 artifacts（空数组合法，比如没有 stable+ 规则）。 */
  compile(entries: KnowledgeEntry[]): SkillArtifact[];
  /** IO：把 artifacts 写到 skills 目录（adapter 实现）。 */
  write(artifacts: SkillArtifact[]): Promise<{ written: string[]; skipped: string[] }>;
  /** IO：删除指定 ruleId 列表的 skill 目录（幂等，ENOENT 静默跳过）。 */
  cleanup(ruleIds: string[]): Promise<{ removed: string[] }>;
}
```

- [ ] **Step 1: 默认目录解析**

```typescript
import os from "node:os";
import path from "node:path";
import fs from "node:fs/promises";
import { formatAsAgentSkill } from "@teamagent/core";

const DEFAULT_DIR = path.join(os.homedir(), ".claude", "skills", "teamagent");
const STABLE_PLUS = ["stable", "canonical", "enforced"] as const;

export interface SkillCompilerOptions {
  skillsDir?: string;                    // 默认读 TEAMAGENT_SKILLS_DIR env，再 fallback DEFAULT_DIR
  now?: () => string;
}

export function makeSkillCompiler(opts: SkillCompilerOptions = {}): SkillCompiler {
  const dir = opts.skillsDir ?? process.env.TEAMAGENT_SKILLS_DIR ?? DEFAULT_DIR;
  return {
    compile(entries) {
      return entries
        .filter((e) => e.status === "active" && STABLE_PLUS.includes(e.current_tier as any))
        .map((e) => ({
          ruleId: e.id,
          dirname: e.id,
          skillMd: formatAsAgentSkill(e),
        }));
    },
    async write(artifacts) {
      const written: string[] = [];
      const skipped: string[] = [];
      for (const a of artifacts) {
        const ruleDir = path.join(dir, a.dirname);
        await fs.mkdir(ruleDir, { recursive: true });
        const filePath = path.join(ruleDir, "SKILL.md");
        // 原子写
        const tmp = `${filePath}.tmp-${process.pid}-${Date.now()}`;
        await fs.writeFile(tmp, a.skillMd, "utf-8");
        await fs.rename(tmp, filePath);
        written.push(a.ruleId);
      }
      return { written, skipped };
    },
    async cleanup(ruleIds) {
      const removed: string[] = [];
      for (const id of ruleIds) {
        const ruleDir = path.join(dir, id);
        try {
          await fs.rm(ruleDir, { recursive: true, force: true });
          removed.push(id);
        } catch {
          // 幂等，忽略
        }
      }
      return { removed };
    },
  };
}
```

## Task 10: 统一 `runCompile(deps)` 入口 + 更新调用方

**目标：** 把过去散落在 analyze/pitfall/calibrate 里的 `new MarkdownCompiler().writeToFile(...)` 统一成一个 `runCompile(deps)` 函数，一次性 write 两个出口 + 处理订阅到的 skill 事件。

**Files:**

**设计：**

```typescript
export interface CompilePipelineDeps {
  store: KnowledgeStore;
  markdownCompiler: Compiler<string> & { writeToFile: (e: KnowledgeEntry[]) => CompileWriteInfo };
  skillCompiler: SkillCompiler;
  bus?: AttributionBus;
  /** 上一轮 calibrate 发出的 skill 事件（可选，fallback 走 "全量 diff"） */
  skillEvents?: { action: "skill_should_write" | "skill_should_remove"; id: string }[];
  dryRun?: boolean;
}

export interface CompilePipelineResult {
  markdown: { path: string; tokens: number; shownCount: number; truncatedCount: number };
  skills: { written: string[]; removed: string[] };
}

export async function runCompile(deps: CompilePipelineDeps): Promise<CompilePipelineResult> {
  const entries = deps.store.getAll();
  // 1. CLAUDE.md：内部会按 canonical+ 过滤 + 2000 token 预算
  const mdInfo = deps.dryRun
    ? { filePath: "(dry-run)", blockLineCount: 0, blockStartLine: 0 }
    : deps.markdownCompiler.writeToFile(entries);

  // 2. Skills：event-driven，有就按事件走；否则全量 diff（比对当前目录和应有状态）
  const artifacts = deps.skillCompiler.compile(entries);
  const { written } = deps.dryRun
    ? { written: artifacts.map((a) => a.ruleId) }
    : await deps.skillCompiler.write(artifacts);

  const toRemove = deps.skillEvents
    ?.filter((e) => e.action === "skill_should_remove")
    .map((e) => e.id) ?? [];
  const { removed } = deps.dryRun
    ? { removed: toRemove }
    : await deps.skillCompiler.cleanup(toRemove);

  deps.bus?.emit({
    source: "compile",
    action: "markdown_compiled",
    target: { id: mdInfo.filePath },
    severity: "info",
    userFacingValue: `wrote ${artifacts.length} skills, ${written.length} skill files, removed ${removed.length}`,
    timestamp: new Date().toISOString(),
  });

  return {
    markdown: { path: mdInfo.filePath, tokens: 0 /* adapter 返回？待定 */, shownCount: 0, truncatedCount: 0 },
    skills: { written, removed },
  };
}
```
