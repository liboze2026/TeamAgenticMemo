## 一、单句目标

> **自动从 5 个源（GitHub releases、npm、RSS、arxiv、手动）抓取前沿知识，经 Haiku 价值判断后存入 SQLite + sqlite-vec，为 M2.7 Inline 注入提供数据基础。**

---

## 二、背景与定位

### RAG 架构位置

M2.6 + M2.7 合起来是 TeamAgent 第一套完整语义 RAG：

```
M2.6 Index:    抓取 → 过滤 → LLM提炼 → 存SQLite → 生成向量
M2.7 Retrieve: UserPromptSubmit → sqlite-vec余弦匹配 → Top-K → 注入上下文
Generate:      Claude Code 本身（注入后由 AI 自主采纳）
```

与现有规则系统（通道 1-3）的区别：规则系统用词法/AST pattern matching，M2.6+M2.7 是第一套**语义向量检索**。

### 与 Phase 2 四通道的关系

| 通道 | 机制 | M2.6/M2.7 关系 |
|------|------|---------------|
| 1 CLAUDE.md | 静态全量注入 | 无关 |
| 2 Agent Skills | pull-based，Claude 自主 invoke | 无关 |
| 3 Hook 拦截 | tool_use 触发，词法匹配 | 无关 |
| **4 Wiki 注入** | UserPromptSubmit → 向量匹配 | **M2.6 = 建索引，M2.7 = 检索注入** |

## 三、设计决策记录

| # | 决策点 | 选择 | 原因 |
|---|--------|------|------|
| D1 | 嵌入模型 | 本地 @xenova/transformers（all-MiniLM-L6-v2） | 本地优先，零 API 成本；**标记为后期可换 Anthropic Embeddings API** |
| D2 | 源范围 | 全 5 源（GitHub releases / npm / RSS / arxiv / 手动） | 完整覆盖，各源独立适配器，后加成本低 |
| D3 | 触发方式 | 手动 `wiki:pull` + 每日 cron，不加 session start hook | session start 加 latency 不划算 |
| D4 | 栈检测 | 自动检测（package.json 等）+ 手动 override 合并 | 自动覆盖 80% 场景，手动处理特殊库 |
| D5 | JS-rendered 页面 | 跳过（记 fetch_error），不引入 Firecrawl | 大多数 changelog 是静态页，Firecrawl 推 Phase 5 |
| D6 | 架构模式 | WikiSourcePort 契约 + SourceRegistry | 与 M2.5-half ErrorSignalCollector 模式一致，Port 先行 |
| D7 | 源订阅方式 | 栈感知自动订阅（首次 wiki:pull 自动检测栈并订阅） | 符合"对用户无需任何主动操作"的产品愿景 |

## 四、整体架构

```
Trigger (manual wiki:pull / daily cron)
        ↓
StackDetector.detect()          ← auto(package.json/requirements.txt/go.mod/Cargo.toml)
        +                          + manual config override
AutoSubscriber.subscribe(stack) ← 首次运行：栈 → 自动订阅对应源
        ↓
SourceRegistry.fetchAll(since)  ← 5个WikiSourcePort适配器并行抓取
        ↓  RawWikiItem[]
WikiFilter.filter(items, stack) ← 纯函数，栈相关性预筛（机械子串，无LLM）
        +  filterByAge(items, maxAgeDays=180)
        ↓  FilteredWikiItem[]
HaikuJudge.judge(items)         ← 批量调用（10条/请求），提取TL;DR + keywords
        ↓  JudgedWikiItem[]
ValidatorL0.validate(items)     ← 机械检查（url合法/tldr非空/keywords非空）
        ↓  ValidWikiItem[]
WikiStore.save(items)           ← 写 knowledge(type=wiki) + wiki_meta，去重幂等
        ↓
EmbedderPort.embed(items)       ← @xenova/transformers，写 knowledge_vec（384维）
        ↓
PipelineReport { added, skipped, rejected, errors }
```

**Functional Core**（`packages/core/`，禁 IO）：
- `detectStack(files: Record<string, string>): string[]`
- `filterByStack(item, stack): boolean`
- `filterByAge(item, maxAgeDays): boolean`
- `buildWikiEntry(judged: JudgedWikiItem): WikiEntry`
- `validateWikiItem(item): ValidationResult`

**Ports**（先写契约测试）：
- `WikiSourcePort` — 每个源的统一抓取接口
- `WikiEmbedderPort` — 嵌入生成接口（D1 标记：后期可换实现）

## 五、WikiSourcePort 接口 + 5 个适配器

### Port 契约

```typescript
interface RawWikiItem {
  sourceType: "github_release" | "npm" | "rss" | "arxiv" | "manual";
  sourceUrl: string;
  title: string;
  content: string;        // 原始正文（readability抓取 or API返回）
  publishedAt: Date;
  sourceId: string;       // 去重键（release tag / npm version / arxiv id 等）
}

interface WikiSourcePort {
  readonly sourceType: RawWikiItem["sourceType"];
  fetch(config: WikiSourceConfig, since: Date): Promise<RawWikiItem[]>;
}
```

**契约测试**（`packages/ports/src/__tests__/wiki-source-contract.ts`）：
- `fetch()` 返回数组（空数组合法，不抛异常）
- 每条 `sourceId` 在同一批次内唯一
- `publishedAt` 早于 `since` 的条目不返回
- `content` 非空字符串
- 网络失败时抛 `WikiFetchError`（不吞异常）

### 5 个适配器

| 适配器 | 抓取方式 | 配置字段 | 特殊处理 |
|--------|---------|---------|---------|
| `GithubReleaseSource` | GitHub releases API（public repo，无需 token） | `repo: string` | 跳过 pre-release（`prerelease: true`） |
| `NpmSource` | npm registry `https://registry.npmjs.org/<pkg>` | `package: string` | 取 changelog 字段 + latest/previous 版本 diff |
| `RssSource` | `rss-parser` Atom/RSS + `@mozilla/readability` 抓正文 | `url: string` | JS-rendered 页面记 `fetch_error` 后跳过 |
| `ArxivSource` | arxiv RSS `export.arxiv.org/rss/<category>` | `category: string` | 只取 abstract，不抓全文；过滤非 CS/AI 类目 |
| `ManualSource` | 用户输入 URL，`@mozilla/readability` 抓正文 | n/a（CLI 直接传入） | 立即入队，不经 cron |

### SourceRegistry

```typescript
class SourceRegistry {
  register(source: WikiSourcePort): void;
  async fetchAll(since: Date): Promise<RawWikiItem[]>;  // 并行调用，单源失败不影响其他
}
```

## 六、StackDetector + AutoSubscriber

### StackDetector（Functional Core，禁 IO）

```typescript
// Shell 层读取文件内容后传入，Core 只做解析
function detectStack(files: Record<string, string>): string[] {
  const stack: string[] = [];

  if (files["package.json"]) {
    const pkg = JSON.parse(files["package.json"]);
    stack.push(...Object.keys(pkg.dependencies ?? {}));
    stack.push(...Object.keys(pkg.devDependencies ?? {}));
  }
  if (files["requirements.txt"])
    stack.push(...parseRequirements(files["requirements.txt"]));
  if (files["go.mod"])
    stack.push(...parseGoMod(files["go.mod"]));
  if (files["Cargo.toml"])
    stack.push(...parseCargoToml(files["Cargo.toml"]));

  return [...new Set(stack)];
}

// Shell 层合并 manual override（来自 teamagent config）
function mergeStack(detected: string[], manual: string[]): string[] {
  return [...new Set([...detected, ...manual])];
}
```

### AutoSubscriber

内置映射表（`packages/core/src/wiki/stack-source-map.ts`），预置 ~50 个常见包：

```typescript
const STACK_TO_SOURCES: Record<string, WikiSourceConfig[]> = {
  "axios":      [{ type: "github_release", repo: "axios/axios" },
                 { type: "npm", package: "axios" }],
  "react":      [{ type: "github_release", repo: "facebook/react" },
                 { type: "rss", url: "https://react.dev/blog/rss.xml" }],
  "typescript": [{ type: "github_release", repo: "microsoft/TypeScript" }],
  "anthropic":  [{ type: "github_release", repo: "anthropics/claude-code" },
                 { type: "rss", url: "https://www.anthropic.com/news/rss.xml" }],
  // ... ~50 个常见包
};

function autoSubscribe(stack: string[]): WikiSourceConfig[] {
  return stack.flatMap(pkg => STACK_TO_SOURCES[pkg] ?? []);
}
```

**触发时机**：首次 `wiki:pull` 时，若 `wiki_subscriptions` 表为空，自动运行 AutoSubscriber。之后 cron/手动拉取都读 `wiki_subscriptions` 表。

## 七、WikiFilter（Functional Core）

```typescript
// 栈相关性预筛（机械子串，无LLM，在 Haiku 判断之前运行）
function filterByStack(item: RawWikiItem, stack: string[]): boolean {
  const text = (item.title + " " + item.content).toLowerCase();
  return stack.some(s => text.includes(s.toLowerCase()));
}

function filterByAge(item: RawWikiItem, maxAgeDays: number = 180): boolean {
  const age = (Date.now() - item.publishedAt.getTime()) / 86400_000;
  return age <= maxAgeDays;
}
```

**设计意图**：预筛减少进入 Haiku 的条目数，降低 API 成本。false positive 由 Haiku 二次过滤，false negative 可接受（宁可漏掉也不要乱投）。

## 八、Haiku 价值判断

```typescript
interface JudgedWikiItem extends RawWikiItem {
  tldr: string;           // 1-2句核心摘要
  keywords: string[];     // 关键词列表（供 M2.7 相似度匹配）
  valuable: boolean;
  rejectReason?: string;
}
```

**Prompt 模板**（批量 10 条/请求）：

```
You are evaluating tech knowledge snippets for a developer knowledge base.
Project stack: {stack.join(", ")}

For each item, decide: is this genuinely useful to a developer using this stack?
Useful = new API, breaking change, deprecation, important bug fix, best practice update.
Not useful = marketing, minor patch notes, unrelated ecosystem, opinion without action.

Return JSON array:
[{
  "sourceId": "...",
  "valuable": true|false,
  "tldr": "1-2 sentences max",
  "keywords": ["keyword1", "keyword2"],
  "rejectReason": "reason or null"
}]

Items:
{items.map(formatItem).join("\n---\n")}
```

**成本估算**：
- 原始条目 100 条 → 预筛后 ~20 条进 Haiku → 2 次 API 调用
- 每次 ~$0.001（Haiku 输入价格）
- **每日总成本 < $0.01**

## 九、ValidatorL0（Functional Core）

```typescript
function validateWikiItem(item: JudgedWikiItem): ValidationResult {
  if (!item.valuable)
    return { valid: false, reason: item.rejectReason ?? "haiku rejected" };
  if (!item.tldr || item.tldr.trim().length < 10)
    return { valid: false, reason: "tldr too short" };
  if (!item.keywords || item.keywords.length === 0)
    return { valid: false, reason: "no keywords" };
  if (!isValidUrl(item.sourceUrl))
    return { valid: false, reason: "invalid url" };
  return { valid: true };
}
```

被拒绝条目写入 `wiki_rejection_log` 表，`teamagent wiki:rejected` 可查看。

## 十、存储模型

```sql
-- Wiki 条目复用 knowledge 表（type='wiki', category='W'）
-- Wiki 专用 metadata
CREATE TABLE wiki_meta (
  knowledge_id             TEXT PRIMARY KEY,
  source_url               TEXT NOT NULL,
  source_type              TEXT NOT NULL,
  source_id                TEXT NOT NULL,
  published_at             TEXT NOT NULL,
  tldr                     TEXT NOT NULL,
  keywords                 TEXT NOT NULL,   -- JSON array
  user_thumbs_down         INTEGER DEFAULT 0,
  inline_injection_count   INTEGER DEFAULT 0,
  fetch_error              TEXT             -- JS-rendered跳过时记录
);

-- 防重复入库
CREATE UNIQUE INDEX idx_wiki_source ON wiki_meta(source_type, source_id);

-- 订阅配置
CREATE TABLE wiki_subscriptions (
  id          TEXT PRIMARY KEY,
  source_type TEXT NOT NULL,
  config      TEXT NOT NULL,   -- JSON，对应各源 config 字段
  auto_added  INTEGER DEFAULT 0,  -- 1=AutoSubscriber自动加，0=用户手动加
  enabled     INTEGER DEFAULT 1,
  created_at  TEXT NOT NULL
);

-- 拒绝日志
CREATE TABLE wiki_rejection_log (
  id          TEXT PRIMARY KEY,
  source_type TEXT,
  source_id   TEXT,
  title       TEXT,
  reason      TEXT,
  rejected_at TEXT
);
```

**嵌入存储**（复用现有 `knowledge_vec` 虚拟表）：
- 嵌入内容：`tldr + " " + keywords.join(" ")`（不嵌全文，控制噪声）
- 维度：384（all-MiniLM-L6-v2）
- 写入：ValidatorL0 通过后异步生成
