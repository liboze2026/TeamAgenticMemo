# TeamAgent Roadmap（最终目标 + 小任务拆分 + 第三方验收）

> 日期：2026-04-30
> 状态：Active

## 最终目标（Goal）

用户无感的上传下载，分享踩坑经验和实用用法，团队里别人踩过的坑后人不再踩。

---

## 小任务拆分（Small Tasks）

### Task 1：跨人同步打通
**目标**：A 的经验可自动传给 B，并在 B 的后续会话里生效。

### Task 2：冲突识别与治理
**目标**：互斥经验可被识别，并有明确裁决结果（保留/合并/下线）。

### Task 3：隐私守门
**目标**：同步前能拦截或脱敏敏感信息（如邮箱、内部主机名、内部路径）。

### Task 4：反噬控制
**目标**：被同步的团队经验不会大量误伤，拒绝/回滚维持在可接受范围。

### Task 5：收益声明（先于验证）
**目标**：先完成可审计的收益声明（Claim），明确定义口径、假设、证据来源与反证条件；暂不把“重复犯错率相对基线下降 ≥ 30%”作为当前验收项。

### Task 6：真实团队稳定运行
**目标**：≥ 3 人连续使用 ≥ 1 周，且 Task 1-5 持续成立。

---

- 进入量化验证条件：连续 2 周数据完整性达标后，再启用“相对下降 ≥ 30%”验收。

## 第三方验收（How to verify from 3rd harness）

> 说明：第三方验收只看“现象与结果”，不看实现细节。

1. **同步验收**：随机抽取 A/B 账号，确认 A 新增经验后，B 在后续会话中可观察到一致效果。
2. **冲突验收**：投放互斥经验样本，确认系统给出明确且可追溯的裁决结果。
3. **隐私验收**：投放含敏感字段样本，确认同步前出现“拦截或脱敏”的可见结果。
4. **稳定性验收**：在自然使用周期内，确认拒绝/回滚保持在预设阈值以内。
5. **收益声明先行**：先输出可审计 Claim（含口径/假设/证据/反证条件）；待实现与数据稳定后再进入收益验收。
6. **团队验收**：至少 3 名成员连续使用 1 周后，再次复核 1-5 条全部成立。

---

## 周度看板

- 上行同步成功率
- 下行同步成功率
- 冲突触发次数 / 平均解决时长
- 隐私拦截命中率 / 漏检数
- 团队规则拒绝/回滚次数
- 重复犯错率（相对基线）

## 出口条件

当“第三方验收”6条连续两个评估周期全部达标，即可宣布完成最终目标。
