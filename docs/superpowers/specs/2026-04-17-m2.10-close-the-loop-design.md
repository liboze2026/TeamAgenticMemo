# M2.10 — Close the Loop：接入 UserPromptSubmit + Stop Hook 自动学习

> 版本: 1.0 | 日期: 2026-04-17 | 状态: 设计完成，待实施

---

## 一、背景与问题

Phase 1 + M2.x 的开发遗留了两个核心偏差，导致系统的两条关键闭环实际上没有打通：

### 偏差 1：UserPromptSubmit Hook 代码写了但没挂上

`packages/cli/src/bin-user-prompt-submit.ts` 已完整实现（关键词提取 → 向量检索 → 频控 → 注入），但：
- `install-hook.ts` 从未注册 `UserPromptSubmit` 分支
- `tsup.hook.config.ts` 产物命名为 `bin-wiki-inject.cjs`（与源文件名不一致）
- 结果：wiki injection pipeline 从未在真实会话中运行过

### 偏差 2：自动学习不自动

系统设计承诺"会话结束后异步运行 PostSession 分析器"，实际上：
- 没有会话结束触发器
- 知识提取靠用户手动运行 `teamagent analyze`
- 不跑就不学，"越用越聪明"的闭环未闭合

---

## 二、目标

1. **UserPromptSubmit Hook 真实运行**：挂上 hook，验证 wiki injection 在真实会话中工作
2. **Stop Hook 自动学习**：会话结束时自动触发 `analyze → calibrate → compile` 流水线
3. **可配置模式**：默认阻塞（用户可见进度），一键切换后台异步

## 三、方案设计

### 3.1 Gap 1：UserPromptSubmit Hook 注册

#### 命名修正

`tsup.hook.config.ts` 显式指定产物名，确保一致性：

```typescript
entry: {
  'bin-pre-tool-use':      'src/bin-pre-tool-use.ts',
  'bin-post-tool-use':     'src/bin-post-tool-use.ts',
  'bin-user-prompt-submit': 'src/bin-user-prompt-submit.ts',
  'bin-stop':              'src/bin-stop.ts',   // M2.10 新增
},
```

产物统一为 `dist/bin-*.cjs`，无歧义。

#### install-hook.ts 新增 UserPromptSubmit 分支

```typescript
const USER_PROMPT_TAG = "teamagent-user-prompt-submit";

// installHook() 新增逻辑：
const userPromptEntry = path.join(cliRoot(), "dist", "bin-user-prompt-submit.cjs");
const hasUserPromptBundle = fs.existsSync(userPromptEntry);

if (hasUserPromptBundle) {
  settings.hooks.UserPromptSubmit ??= [];
  const existing = settings.hooks.UserPromptSubmit.find(
    h => h._teamagentTag === USER_PROMPT_TAG
  );
  if (!existing) {
    settings.hooks.UserPromptSubmit.push({
      _teamagentTag: USER_PROMPT_TAG,
      hooks: [{
        type: "command",
        command: `node ${shellQuote(toForwardSlash(userPromptEntry))}`,
        timeout: 10,   // bin 内部已有 5s 超时，外层留余量
      }]
    });
  }
}
```

**注意**：UserPromptSubmit hook 无 `matcher` 字段（对所有用户消息触发）。

#### uninstallHook() 同步清理

```typescript
if (settings.hooks.UserPromptSubmit) {
  settings.hooks.UserPromptSubmit = settings.hooks.UserPromptSubmit.filter(
    h => h._teamagentTag !== USER_PROMPT_TAG
  );
  if (settings.hooks.UserPromptSubmit.length === 0)
    delete settings.hooks.UserPromptSubmit;
}
```

#### 验收标准

发一条包含项目相关关键词的消息，Claude Code 收到的 prompt 中出现 `[TeamAgent Wiki]` 注入片段。

### 3.2 Gap 2：Stop Hook 自动学习

#### 数据流

```
Claude Code 会话结束
    ↓
Stop hook 触发
input: { session_id, transcript_path, cwd, hook_event_name: "Stop" }
    ↓
bin-stop.ts 读取 .teamagent/config.json → stop_mode
    ├─ sync（默认）: 串行跑流水线，statusMessage 显示进度，Claude Code 等待
    └─ async: spawn detached 子进程后立即 exit 0，Claude Code 立即关闭
    ↓
流水线（pipeline-stop.ts）：
  Step 1: analyze --session-file <transcript_path>
          → 纠正时刻识别 → 新知识条目写入 DB
  Step 2: calibrate
          → 置信度校准 → tier 升降
  Step 3: compile
          → 重新生成 CLAUDE.md TeamAgent 区块
```

#### 新文件：`packages/cli/src/bin-stop.ts`

```typescript
// stdin: StopHookInput { session_id, transcript_path, cwd, hook_event_name }
// 职责：读配置 → 按模式执行流水线 → 任何错误 exit 0
// 绝不能 exit 非 0（会阻断会话关闭）

const STOP_PIPELINE_TIMEOUT_MS = 55_000; // 留 5s 给 hook timeout=60 的边界

async function main(): Promise<void> {
  const input = readStdinJson() as StopHookInput;
  const config = readTeamAgentConfig(input.cwd);

  if (config.stop_mode === "async") {
    spawnDetached(input);   // 立即返回
    return;
  }

  // sync 模式
  await Promise.race([
    runStopPipeline(input),
    timeout(STOP_PIPELINE_TIMEOUT_MS),
  ]);
}

main().catch(() => {
  appendErrorLog("~/.teamagent/stop-errors.log", ...);
  process.exit(0);  // 绝不阻断
});
```

#### sync 模式 statusMessage（Claude Code 界面显示）

Stop hook 输出到 stderr 的内容会作为 statusMessage 展示：

```
TeamAgent: 分析会话中...
TeamAgent: 发现 2 个纠正时刻，写入 2 条新知识
TeamAgent: 校准完成，3 条规则置信度更新
TeamAgent: CLAUDE.md 已更新，当前 18 条活跃知识
```

#### async 模式：detached spawn

```typescript
function spawnDetached(input: StopHookInput): void {
  const child = spawn(process.execPath, [stopPipelineScript, JSON.stringify(input)], {
    detached: true,
    stdio: "ignore",
    cwd: input.cwd,
  });
  child.unref();
}
```

Windows 兼容：`detached: true` + `stdio: "ignore"` + `child.unref()` 三件套。

#### config 命令

```bash
pnpm teamagent config stop-mode sync    # 默认
pnpm teamagent config stop-mode async   # 切后台
pnpm teamagent config show              # 查看当前所有配置
```

**`.teamagent/config.json` 结构**：

```json
{
  "stop_mode": "sync"
}
```

文件不存在时默认 `sync`。不入 git（加入 `.gitignore`）。

#### install-hook.ts 新增 Stop 分支

```typescript
const STOP_HOOK_TAG = "teamagent-stop";

// Stop hook 无 matcher 字段（Claude Code 规范）
settings.hooks.Stop ??= [];

    }]

```

## 四、错误处理原则

| 场景 | 处理方式 |
|------|---------|
| Stop hook 任何异常 | catch → 写 `~/.teamagent/stop-errors.log` → exit 0 |
| analyze 找不到 transcript | 跳过 analyze，继续 calibrate + compile |
| calibrate 失败 | 记录，继续 compile |
| compile 失败 | 记录，流水线结束 |
| 超过 55s 超时 | 立即 exit 0，不等待 |
| UserPromptSubmit 任何异常 | exit 0（现有逻辑已有保障） |

**核心原则：Stop hook 绝不能阻断会话关闭。**

## 五、退出标准

1. **UserPromptSubmit 验收**：真实会话中发含项目关键词的消息，AI 收到的 prompt 包含 `[TeamAgent Wiki]` 注入片段
2. **Stop hook sync 验收**：会话结束时 Claude Code 界面显示 "TeamAgent: ..." 进度，结束后 CLAUDE.md 已更新
3. **Stop hook async 验收**：`teamagent config stop-mode async` 后，会话关闭无延迟，知识库在后台更新
4. **幂等安装**：`teamagent install-hook` 重复执行不产生重复 hook 条目
5. **卸载干净**：`teamagent uninstall` 清除 UserPromptSubmit + Stop 两条 hook
6. **无 runtime crash**：连续 5 次会话，`~/.teamagent/stop-errors.log` 无新增错误
