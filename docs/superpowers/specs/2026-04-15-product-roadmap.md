## 一、产品愿景（用户语，不改）

> 开发出一款类似**团队智脑**的自进化 AI。最终目的是让团队有人踩过的坑只踩一次、别人不再踩；走过的弯路只走一次、别人不再走；有最新的知识和经验都**实时同步**到团队中每一个人的 Claude Code 里。

**核心机制**：系统在运行中自己完善自己，实现**自我进化**。对用户**无需任何主动操作**——系统自动记住错误、自动记住经验、自动把用户产生的知识上传到团队。整个系统**越用越聪明、越快、错误越少、token 消耗越少**。

---

## 二、系统的 5 个维度（坐标系）

愿景展开后，系统要在 5 个正交轴上扩张：

| 轴 | 含义 | Phase 1 状态 | 终态 |
|---|---|---|---|
| **知识类型** | wiki / taste / 前沿 + 成功经验 + 失败经验 | ❌ 只有失败经验（避坑） | wiki + 成功 + 失败 |
| **知识来源** | 个人 / 团队 / 互联网 | ❌ 只有个人 | 三源融合 |
| **激活方式** | 主动补充（对话中主动 mention）+ 被动运用（运行时拦截） | ❌ 只有被动 | 主动 + 被动 |
| **工具范围** | Claude Code / Cursor / Windsurf / 其它 | ✓ Claude Code | 多工具 |
| **用户范围** | 本地单人 / 团队共享 | ✓ 本地单人 | 团队 |

**Phase 1 = 5D 空间的一个角落**（失败经验 × 个人知识 × 被动 × Claude Code × 本地单人）。后续每个 Phase 扩张一个轴或深化一个点。

---

## 三、自我进化的四个子能力

"自我进化"不是一个 feature，是贯穿 5 轴的**元能力**。它由 4 个子能力组成：

1. **自动记忆错误** — 捕获纠正时刻并抽成规则（Phase 1 M3/M4 已具备）
2. **自动记忆经验** — 同上，但针对成功模式（Phase 1 M3 有框架，Phase 2 才真落地）
3. **自动置信度管理** — Calibrator 根据使用情况上下调、over-archive 保护（Phase 1 M6 + Stage A，Phase 2 深化）
4. **自动知识传播** — 高置信个人知识 → 自动提议同步到团队（Phase 4 才做）

"越用 token 越少"是以上 4 个能力跑起来后的**自然结果**（规则更准 → AI 少走弯路 → 少消耗）——不是单独一个子系统。

## 四、6 个 Phase 分段（纵深打法）

**原则**：一个 Phase 把一个轴推到尽头或一个角落深挖到极致，不横展铺摊。

### Phase 1 — Walking Skeleton（已完成，2026-04-15）

**角落**：失败经验 × 个人知识 × 被动 × Claude Code × 本地单人  
**版本**：v0.1.0  
**产出**：13/13 设计承诺交付；32 条活跃知识；168 次真实 Hook 拦截；Calibrator 自修正闭环已跑通  
**遗留**：所有"已知漏洞"记录在 `docs/specs/2026-04-15-phase2-backlog.md`（30+ 条）

### Phase 2 — 把本地单用户做到极致 + 证明价值

**目标**：深挖当前的角落，把单用户本地的体验做到无可挑剔，并用**硬数据**证明"装 vs 不装"有显著差距。

**覆盖轴**：
- **工具范围 / 用户范围 / 知识来源：不动**（所有"扩张"留给 Phase 3+）
- 知识类型：从"只失败经验" → **"失败 + 成功"**（Phase 1 M3 框架真落地）
- 激活方式：从"只被动" → **"被动 + 主动 v1"**（session 开场简报 + on-demand brief）

**核心产出**（详细 sub-project 见 `2026-04-15-phase2-design.md`）：
- Benchmark 基础设施 + AI override signal → 能量化"装 vs 不装"
- Calibrator 深化（时间衰减、更细信号、over-archive 保护）
- Scope intelligence（LLM 推断 project-aware scope）
- Matcher 硬化（区分"使用 X" vs "讨论 X"）
- Detector 升级 + 成功经验真落地
- 主动知识补充 v1（session 开场简报）
- Hook 韧性 + 运维基础（bundle 自动重建、events 轮转、PII 过滤）

**退出标准**：benchmark 数据证明有意义的重复犯错率下降；作者连续用 1 个月无"规则反噬"事件。

**版本**：v0.3.0

### Phase 3 — Ready for User #2（第一次跨越作者）

**目标**：**用户范围**从"只有作者" → **"任何一个朋友能自己装上"**。

**覆盖轴**：
- 用户范围：本地单人 → 本地少数人（各自单独使用，未联通）
- 其它轴：不动

**核心产出**：
- `teamagent init` 在陌生项目跑通（非 teamagent 仓库）
- `teamagent doctor` 诊断安装问题
- 错误消息友好化（Phase 1 大量直接抛栈）
- README + 5 分钟上手指南
- 跨平台充分测试（macOS / Linux）
- 邀请 2-3 个朋友实测，收集反馈并迭代

**退出标准**：至少 3 位非作者朋友独立装上并持续使用 ≥ 2 周。

**版本**：v0.5.0

### Phase 4 — 团队层真正落地

**目标**：**用户范围**从"几个独立使用的朋友" → **"同一团队共享知识"**；**知识来源**加入"团队"维度。

**覆盖轴**：
- 用户范围：独立少数人 → 团队
- 知识来源：只个人 → 个人 + 团队

**核心产出**：
- Git-backed 知识同步（`scope=team` / `scope=global` 真正工作）
- 冲突处理（两人规则矛盾时如何选择）
- 隐私 / 敏感信息自动过滤（上传到团队前自动 scrub）
- 团队仪表盘（谁贡献了什么、谁受益于什么）
- 自动上传机制（个人高置信规则 → 自动提议同步到团队）
- 团队成员加入/退出流程

**退出标准**：5-10 人真实团队共用 ≥ 1 月，"有人踩过的坑别人不再踩"可验证。

**版本**：v0.7.0

### Phase 5 — 前沿知识 + 主动补充深化

**目标**：**知识来源**加入"互联网"；**激活方式**深化"主动补充"从 v1 到 v2（语义上下文感知）；**知识类型**加入 wiki / taste。

**覆盖轴**：
- 知识来源：个人 + 团队 → 个人 + 团队 + 互联网
- 激活方式：v1 简单版 → v2 语义感知
- 知识类型：失败 + 成功 → 失败 + 成功 + wiki/taste

**核心产出**：
- RAG 前沿技术（paper / blog / 官方文档新特性的自动抓取）
- Tech taste 抽取（不只"别用 X"，还包括"现在最佳实践是 Y"）
- Matcher v3：全语义相似度（向量嵌入）
- 主动补充 v2：根据对话上下文判断**何时**介入、介入**什么内容**
- 预置领域知识包（TypeScript / React / Python 等）

**退出标准**：系统不仅记住团队走过的路，还能带团队去走别人走过的更好的路；用户报告学到了原本不知道的最新技术。

**版本**：v0.9.0

### Phase 6 — 多 AI 工具（MCP）

**目标**：**工具范围**从 Claude Code → **多工具**。

**覆盖轴**：
- 工具范围：Claude Code → Claude Code + Cursor + Windsurf + ...

**核心产出**：
- MCP server（Model Context Protocol）
- 工具间知识共享（一个人用 Cursor 产生的知识能自动同步到 Claude Code）
- 统一事件模型（跨工具的 PersistedEvent）

**退出标准**：至少 2 个非 Claude Code 的 AI 工具接入并真实使用。

**版本**：v1.0.0（公开发布）

## 五、Phase 之间的关键依赖

```
Phase 1 (done)
    ↓
Phase 2 (benchmark 基础设施是 Phase 3+ 所有 A/B 判断的尺子)
    ↓
Phase 3 (第二个用户为 Phase 4 的"团队"铺垫)
    ↓
Phase 4 (团队层为 Phase 5 的"前沿知识注入团队"铺垫)
    ↓
Phase 5 (知识类型+来源扩展)
    ↓
Phase 6 (工具扩展，靠前面所有 Phase 的稳定性支撑)
```

**关键承诺**：每个 Phase 的 benchmark 数据对比**前一个 Phase**，用同一把尺子验证"有实质进步"。

## 六、不在本 Roadmap 范围内的事

- **商业化 / 定价 / 企业版本**：Phase 6 后再考虑
- **云端托管服务**：本系统设计为本地优先，云同步只用 Git，不建后端
- **IDE 插件**：只做 MCP server，不单独做 IDE 原生插件
- **UI / 仪表盘**（除 Phase 4 的极简团队看板外）：CLI + Markdown 即所有界面
- **Windows / Linux / macOS 之外的平台**（如 WSL、BSD）：Phase 6 之后再评估
