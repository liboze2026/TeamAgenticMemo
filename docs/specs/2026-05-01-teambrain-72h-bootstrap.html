// 02 · the progression

Hour 0 → 72.

HOUR 0–2 KICKOFF PHASE 01 / 07
Set the truth on the table.
Human · Owner
  • say what TeamBrain is
  • say what it is not — not a doc site for humans
  • dump every historical pitfall, in writing
HOUR 2–6 PARALLEL BUILD PHASE 02 / 07
Four agents, four lanes, no overlap.
Agent A · Claude
  • scaffold the repo skeleton
  • write README.md
Agent B · Claude
  • write TRAPS.md
  • nail the trap entry format
Agent C · Codex
  • write TASK_TEMPLATE.md
  • write VERIFY_TEMPLATE.md
Agent D · Codex
  • write agent_rules/claude.md
  • write agent_rules/codex.md
HOUR 6–12 REVIEW PHASE 03 / 07
Squeeze out the soft rules.
Reviewer · Gatekeeper
  • flag every vague rule
  • hunt mock loopholes
  • confirm every task is ground-truth verifiable
Human · Owner
  • delete the filler
  • keep only hard rules
  • add real failure cases — by name
HOUR 12–24 FIELD TEST · #1 PHASE 04 / 07
Run a real task. Watch what breaks.
Claude / Codex · in the field
  • read TeamBrain cold
  • execute one real task end-to-end
  • capture evidence at every step
  • mark every error agents still make
HOUR 24–36 PATCH PHASE 05 / 07
Convert pain into rules.
Maintainers · all hands
  • turn each new failure into a trap entry
  • upgrade fuzzy rules into hard rules
  • add the no-mocking checklist
  • add the evidence checklist
HOUR 36–60 FIELD TEST · #2 PHASE 06 / 07
Different task. Different agent.
Fresh agent · clean read
  • swap in a new task — not the same one
  • swap in a different agent — no shared memory
  • verify the patches actually prevent Task #1's errors
HOUR 60–72 RELEASE PHASE 07 / 07
Cut v0.1.
Owner · ship
  • clean the directory tree
  • write the onboarding flow
  • write usage examples
  • tag the release
