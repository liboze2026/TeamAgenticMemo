| File | Upstream | Patched | Reason |
|---|---|---|---|
| `bin/gstack-team-init` | JS source writes the hook command with an escaped `$` in the string literal | builds the `$CLAUDE_PROJECT_DIR` command using `String.fromCharCode(36)` | Required-mode hook settings must contain an expandable `$CLAUDE_PROJECT_DIR`, not a literal escaped dollar that bypasses enforcement. |

## Not vendored (referenced but missing from upstream bin/)

The following `gstack-*` commands are referenced by SKILL.md or vendored binaries
but do NOT exist as files in `garrytan/gstack:bin/`. They are runtime aliases
generated by gstack's installer. Calls to them are wrapped in
`2>/dev/null || true`, so they degrade gracefully:

`gstack-brain-remote`, `gstack-context`, `gstack-upgrade`,
`gstack-verify-desktop`, `gstack-verify-mobile`, `gstack-verify-tablet`,
`gstack-qa`, `gstack-ship`, `gstack-question-sensitivity`.

The SKILL.md preambles resolve `GSTACK_BIN` at runtime. Project-level mirrors are
authoritative: Codex prefers `.codex/skills/gstack/bin`, Claude prefers
`.claude/skills/gstack/bin`, and user-level installs are fallback only.

## Update procedure

```bash
SHA=<new-upstream-sha>
while IFS= read -r b; do
  curl -fsSL "https://raw.githubusercontent.com/garrytan/gstack/$SHA/bin/$b" \
    -o ".claude/skills/gstack/bin/$b"
  cp ".claude/skills/gstack/bin/$b" ".codex/skills/gstack/bin/$b"
done <<'BINS'
gstack-brain-enqueue
```
